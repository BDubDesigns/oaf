# OAF Agent Guidance

OAF works only on greenfield apps generated by `oaf init`. Treat the generated
app layout and `oaf/` metadata as product contracts, not suggestions.

Use the fixed OAF tool set only: `read`, `list`, `grep`, `write`, and
`command`. Do not discover or install arbitrary tools. Do not invent new
top-level application structure.

Read the app markers and local context before changing files. Keep work small,
state assumptions clearly, and leave dependency, schema, and metadata changes
to their explicit policies. Meaningful future runs will produce receipts.
